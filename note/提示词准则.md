# Prompt Engineering 技术比较笔记

## 一、概念与核心思路

### 1. CoT（Chain of Thought）

**核心目标：** 让模型显式展示中间推理步骤，从而提升逻辑严谨性与任务可解释性。
 **方法特点：**

- 要求模型在给出最终答案前输出推理链。
- 常用于数学、逻辑推理、复杂指令分解。
- 结构通常为：观察 → 推理步骤 → 最终决定。

### 2. ToT（Tree of Thoughts）

**核心目标：** 让模型并行或分支地生成多路径思考，并对路径进行自我评估后选择最优方案。
 **方法特点：**

- 比 CoT 更复杂，强调“多方案探索”。
- 适用于需要策略搜索、规划、多步骤创造性任务。
- 常见流程：
  - 生成若干思路分支
  - 对每个分支进行评分
  - 选择继续拓展或剪枝
  - 最终选定最优解路径

### 3. Role Playing（角色设定）

**核心目标：** 通过设定明确身份，让模型以该角色的知识体系、风格与偏好执行任务。
 **方法特点：**

- 强调上下文一致性与领域专注
- 常用于模拟专家、导师、顾问、系统组件
- 可与 CoT/ToT 叠加使用以提升策略性或专业度

------

## 二、适用场景对比

| 技术             | 适用问题类型                                     | 示例任务                          | 典型收益                         |
| ---------------- | ------------------------------------------------ | --------------------------------- | -------------------------------- |
| **CoT**          | 明确逻辑链条、规则推理、数学、分析题、结构化分解 | 解方程、推理题、拆解任务步骤      | 让模型逻辑更清晰，减少跳步、错因 |
| **ToT**          | 需要多方向探索、创造性规划、优化决策             | 文案创作、长期规划、算法策略搜索  | 能避开局部最优，提高最终方案质量 |
| **Role Playing** | 专业领域任务、风格控制、对齐某类专家的回答方式   | 扮演 C++ 专家、法律顾问、科研助理 | 输出风格稳定、领域知识响应更准   |

## 三、优势与局限

### CoT

**优势：**

- 强化逻辑清晰度
- 提升对复杂任务的稳定性
- 输出可解释

**局限：**

- 在创造性任务中可能拖慢节奏
- 如果要求太明确，可能生成冗长推理

### ToT

**优势：**

- 多路径探索带来更优质量
- 适合开放式任务
- 能自我反思与修正

**局限：**

- 成本高（token、时间消耗大）
- 对提示工程要求更高
- 可能出现复杂度过高导致任务偏离主题

### Role Playing

**优势：**

- 输出风格稳定、专业度高
- 有利于强化领域知识表达
- 可作为其他技术的基础层

**局限：**

- 角色设定不当可能导致幻觉增强
- 复杂角色设定难以保持长期一致性

## 四、组合策略（实际应用价值高）

### 1. Role Playing + CoT

适合：专业领域推理任务
 示例：

- 设定“你是资深 C++ 编译器工程师”，并要求其在回答前给出逻辑推理链。
   收益：专业性强且思路透明。

### 2. Role Playing + ToT

适合：需要专家探索多方案的创造性任务
 示例：

- “你是资深系统架构师，请基于多个技术路线生成架构决策树并评分”
   收益：结果可控、方案质量高。

### 3. CoT + ToT

适合：既要逻辑严密又需要多方案探索
 示例：

- 数学、高等推理、科研方案探索
   收益：减少错误，多路径验证。

## 五、实际提示模板示例（简洁版）

### CoT 模板

```
请在得出最终答案之前展示完整推理过程。
```

### ToT 模板

```
请生成若干不同方向的解题方案，并对每个方案进行可行性评估。基于评估选择最佳方案并给出最终答案。
```

### Role Playing 模板

```
请以[身份/角色]的视角进行分析，并保持该身份的知识体系和表达方式。
```

### 组合模板示例（复合型）

```
你现在是一名[角色]。请生成至少三个备选方案，并展示每个方案的思路与优缺点。最后基于比较给出你的最佳选择。
```

## 六、总结

- **CoT 解决“思路清晰”问题。**
- **ToT 解决“方案最优”问题。**
- **Role Playing 解决“专业性与一致性”问题。**
- 三者可分离使用，也可以复合使用以强化控制与质量。
