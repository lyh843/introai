# 人工智能导论：对抗搜索 知识点总结

## 一、对抗搜索基础概念

### 1. 核心定义

* **对抗搜索（Adversarial Search）**：又称博弈搜索（Game Search），研究多个智能体（Agent）在竞争环境中交替行动时的决策优化问题。

* **双人零和博弈**：核心假设为信息确定、全局可观察、双方轮流行动，且一方收益必然等于另一方损失（无中间收益），是对抗搜索的核心研究场景。

### 2. 博弈分类维度

* 确定性 / 随机性：行动结果是否可预测。

* 信息完整性：是否所有玩家都掌握全局状态信息。

* 玩家数量：双人 / 多人博弈。

* 收益特性：零和（输赢对立）/ 非零和（可双赢 / 双输）。

### 3. 博弈问题形式化定义

| 要素           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| 状态（State）  | 包含当前游戏局面和当前行动的玩家                             |
| 动作（Action） | 玩家在当前状态下可执行的操作集合，记为`actions(s)`           |
| 状态转移       | 由转移函数`result(s, a)`定义，输入状态`s`和动作`a`，输出后继状态 |
| 终局检测       | 函数`terminal_test(s)`判断游戏是否结束                       |
| 终局得分       | 函数`utility(s, p)`表示终局状态`s`下玩家`p`的收益值          |

### 4. 生成对抗网络（Generative Adversarial Networks, GAN）

构建出一个模型$G:\mathcal{Z} \rightarrow \mathcal{X}$，使之可以生成符合数据分布$p(x)$的样本。

- 生成对抗网络：通过对抗训练和最大最小优化的方式使得生成网络产生的样本服从真实数据分布
- 判别网络：尽量精准地判断一个样本是来自真实数据还是由生成网络产生$D(x, \phi) \rightarrow 1/0$
- 生成网络：尽量生成判别网络无法区分来源的样本$\mathcal{z} \sim \mathcal{N}(0, I)

## 二、确定性搜索算法

### 1. 最大最小搜索（MinMax Search）

#### 单一Agent搜索树

- **状态的价值$v$** 从当前状态出发能获得的最大价值。
- **Agent决策结点：** $V(s) = \max_{s' \in successors(s)} V(s')$
- **对手决策结点：**$V(s) = \max_{s' \in successors(s)} V(s')$

#### 性能分析

* 时间复杂度：`O(b^m)`（`b`为分支因子，`m`为搜索深度）。

* 空间复杂度：`O(bm)`（与 DFS 类似，递归栈存储路径）。

* 局限：对于象棋（`b≈35`，`m≈100`）等复杂博弈，精确搜索不可行。

### 2. Alpha-Beta 剪枝

#### 核心优化目标

在不影响最终决策的前提下，剪去博弈树中无需探索的分支，减少搜索量。

#### 关键参数定义

* `α`（Alpha）：当前路径上 MAX 玩家的最佳（极大值）收益（“至少能获得”）。

* `β`（Beta）：当前路径上 MIN 玩家的最佳（极小值）收益（“至多能给出”）。

#### 剪枝规则

* **对 MAX 节点：**如果其孩子结点（MIN结点）的收益大于当前的$\alpha$值，则将$\alpha$值更新为该收益；
* **对 MIN 节点：**如果其孩子结点（MAX结点）的收益小于当前的$\beta$值1，则将$\beta$值更新为该收益。
* **根节点（MAX结点）：**的$\alpha$值和$\beta$值分别被初始化为$-\infty$和$+\infty$

#### 更新规则

每个节点的$[\alpha, \beta]$从**其父节点提供的初始值**开始，取值按照如下形式变化：

1. $\alpha$逐渐增加，$\beta$逐渐减少
2. 如果一个节点的$\alpha$值和$\beta$值满足$\alpha > \beta$的条件，则该结点尚未被访问的后续节点就会被剪枝，因而不会被访问。

#### 重要说明

* 剪枝效果依赖节点探索顺序，最优顺序下可将时间复杂度降至`O(b^(m/2))`。
* 适用于分支因子较小、评估函数易设计的博弈（如象棋），对围棋等分支因子极大的博弈仍不适用。

#### 代码实现

```python
def alpha_beta(node, depth, α, β, maximizingPlayer):
    if depth == 0 or node is terminal:
        return heuristic_value(node)
    if maximizingPlayer:
        value = -∞
        for each child of node:
            value = max(value, alpha_beta(child, depth-1, α, β, false))
            α = max(α, value)
            if α >= β:
                break   // β 剪枝（剪掉后续兄弟）
        return value
    else:
        value = +∞
        for each child of node:
            value = min(value, alpha_beta(child, depth-1, α, β, true))
            β = min(β, value)
            if α >= β:
                break   // α 剪枝
        return value
```

### 3. 评估函数补充

* 作用：将非终局状态转化为可量化的价值（替代递归到底），核心形式为加权线性函数：

  `Eval(s) = w₁f₁(s) + w₂f₂(s) + ... + wₙfₙ(s)`

  （`wᵢ`为特征权重，`fᵢ(s)`为状态特征，如象棋中 “兵的数量”“车的数量”）。

* 局限：围棋等博弈的状态特征复杂，难以设计有效的评估函数。

## 三、基于模拟的搜索：蒙特卡洛树搜索（MCTS）

### 1. 核心思想

无需复杂评估函数，通过大量随机模拟（自我博弈）估计节点价值，结合 “探索 - 利用” 策略平衡搜索效率与准确性。

### 2. 基础理论：多臂老虎机问题

* 核心矛盾：探索（尝试未知动作以获取更优信息）与利用（选择当前已知最优动作）的平衡。
  - 探索：尝试更多可能得决策，从而估计不同摇臂的优劣
  - 利用：选择当前最优的摇臂
* 经典解决策略：


  * `ε-贪心算法`：以`1-ε`概率利用最优动作，`ε`概率随机探索（未考虑探索次数）。

  * `Softmax`：基于动作平均奖赏分配选择概率，奖赏越高选择概率越大。

  * `UCB（上限置信区间）`：优先选择 “估计价值 + 置信偏差” 最大的动作，平衡探索次数与收益。

### 3. MCTS 四步核心流程

#### （1）选择（Selection）

从根节点出发，按 UCB 等策略向下遍历，直至到达未完全扩展的叶子节点（优先选择 “价值高 + 探索少” 的节点）。

#### （2）扩展（Expansion）

为叶子节点生成一个或多个子节点（对应可行动作），扩展博弈树。

#### （3）模拟（Simulation）

从扩展出的子节点开始，随机执行动作（或按简单策略）模拟博弈至终局，记录模拟结果（胜 / 负）。

#### （4）回溯（Backpropagation）

将模拟结果沿路径反向更新所有节点的统计信息：

* 胜方节点：获胜次数`V`和访问次数`T`均递增。

* 败方节点：仅访问次数`T`递增。

### 4. 关键优化：AlphaGo/AlphaZero 的融合方案

MCTS 与深度学习、强化学习结合，突破传统 MCTS 局限：

#### （1）网络架构

* 策略网络（Policy Network）：`π(a|s;θ)`，输入状态`s`，输出动作概率分布，用于指导 MCTS 的选择与扩展（替代随机选择）。

* 价值网络（Value Network）：`v(s;w)`，输入状态`s`，输出状态价值估计，辅助模拟结果评估（减少模拟次数）。

#### （2）训练流程

* AlphaGo（2016）：先从人类棋谱学习策略网络，再通过自我博弈优化策略与价值网络。

* AlphaZero（无人类知识）：直接通过 MCTS 自我博弈生成训练数据，让策略网络拟合 MCTS 的动作选择概率，价值网络拟合终局收益。

#### （3）MCTS 流程优化

1. 选择：给定状态$s_t$，找出胜算较大的动作，只搜索这些好的动作
   $$
   score(a) = Q(a) + \eta \cdot \frac{\pi(a | s_t ; \theta)}{1 + N(a)}
   $$

   - $Q(a)$：蒙特卡洛搜索模拟带来的动作价值估计
   - $\pi(a |s_t; \theta)$：策略网络的输出
   - $N(a)$：$s_t$下动作$a$已经被访问的次数

2. 扩展：基于策略网络采样对手动作。

3. 评估：`V = 0.5·v(s;w) + 0.5·r_T`（`r_T`为模拟终局收益）。

4. 回溯：平均多次模拟结果更新`Q(a)`。
