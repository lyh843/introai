## 一、题目1

### 已知条件：

#### `mdp`

1. `mdp.getStates()`：
   - 返回值：`list`
   - 返回MDP中所有状态的列表（不包括墙壁）
2. `mdp.getPossibleActions(state)`：
   - 返回值：`list`
   - 返回当前状态的所有可能动作的列表（如果是`terminal`状态将会返回空列表）
3. `mdp.getTransitionStatesAndProbs(state, action)`：
   - 返回值：`(nextState, probb)`对的`list`
   - 表示从当前状态采取`actions`，到不同`nextState`的不同`probb`
4. `mdp.getReward(state, action, nextState)`
   - 返回值：`reward`
   - 返回从`state`采取`action`到达`nextState`所获得的奖励
5. `mdp.isTerminal(state)`
   - 返回值：`bool`
   - 返回当前是否`Terminal`状态（终点）
   - 通常最终状态没有任何未来回报

#### `ValueEstimationAgent`

1. `alpha`： 学习率
2. `epsilon`：探索率
3. `discount`：折扣因子
4. `numTraining`：训练的周期数

#### `gridworld`

1. `getStartState`

   获得起始状态

### 作答思路：

- 对于`computeQValueFromValues(state, action)`函数：

  该函数计算$Q^{\pi *}(s, a) = \sum_{s^* \in S} P_{sa}(s')(r_{sa}(s') + \gamma V^{\pi}(s'))$

- 对于`runvalueIteration`函数：

  该函数实现价值迭代：$V^*(s) = max_a Q^{\pi *}(sa)$

## 二、题目二

**命令：**

```bash
python gridworld.py -g DiscountGrid -a value --discount [你的折扣率] --noise [你的噪声] --livingReward [你的生存奖励]
```

### 大致分析

1. `discount`：即$\gamma$是对于后续或未来价值的一个权重，数值应在$0 \sim 1$之间。
   - 当数值趋于0的时候，可以理解为这个模型几乎不在乎将来的打算，类似于贪心算法，只关注下一步。
   - 当数值趋于1的时候，可以理解为这个模型十分具有远见性，并且把未来的价值与当下的价值看得同等重要
2. `noise`：即模型执行`action`时，不成功执行的概率。也可以代表这个模型的探索度。
   - 当数值趋于0的时候，这个模型会按部就班地按照策略来采取行动。
   - 当数值趋于1的时候，这个模型会几乎随机探索，不按照策略采取行动。可以采取这点让模型失足掉落悬崖，从而让模型规避悬崖。
3. `livingReward`：即模型每走一步时，获得的奖励。
   - 当数值为较小的负值时，模型尽快地向终点奔去
   - 当数值为较大的负值时，模型会直接选择跳下悬崖（
   - 当数值为较小的正值时，会鼓励模型多走几步，向`10`走去
   - 当数值为较大的正值时，在地图里瞎走的奖励比到达终点好多了。

### 1. 偏好近处出口，冒险走悬崖

参数：（0.1，0，-2）

> “待得有点辛苦，但是又不至于去自杀。未来也没啥期望，还是早点逃出去吧”

`discount`设为$0.1$这样一个较小值，使得模型没有什么元件，对比较遥远的`10`没有什么期望。

`livingReward`设为$-2$，使得模型没走一步都会获得惩罚，但是惩罚不至于过重使得模型直接跳下悬崖，逼迫着模型尽快的走一条短路去到出口。

`noise`设为$0$，使得模型按部就班地采取行动，计划走哪就是走哪，不会有偏差。

### 2. 偏好近处出口，但避开悬崖

参数：（0.2， 0.2， -2）

>  “待得有点辛苦，但是又不至于去自杀。未来也没啥期望，得尽可能早点逃出去，但是脚底打滑，得走条安全的路“

与情况一的主要区别在于把`noise`设为了$0.2$，使得模型在走悬崖的时候可能存在失足落崖的可能，从而实现避开悬崖。

此外对比情况一中`discount`为0.1，在这个情况中，`discount`为$0.1$在这里会形成一个小回路，陷入模型的死循环，因此调大一些，让模型能”看到“远处出口的奖励，从而使模型向着奖励奔去。

### 3. 偏好远处出口，冒险走悬崖

参数：（0.5，0，-0.1）

> ”远处的奖励太诱人了，直奔远处出口！“

在这个情况中，由于需要达到远处的出口，因此将`discount`设得比前面两种情况更大，让模型能够看到远处的奖励，使得模型往远处的出口去。

由于出口确实远了点，因此`livingReward`的值也需要相应减小，鼓励模型多走几步，而不是急着离开。

### 4. 偏好远处出口，但避开悬崖

参数：（0.5，0.2，-0.1）

相比情况三，则依旧是加入了“脚滑”这一因素，使得模型需要规避悬崖。

### 5. 避开所有出口和悬崖

参数：（0.5，0，100）

> “待这里不比出去好多了”

将`livingReward`调至了$100$，并且`noise`为$0$，因此这个模型会始终向着$Q$最大的地方走。而且`discount`设为$0.5$使得模型能够判断未来继续瞎逛比走进出口要更好。
